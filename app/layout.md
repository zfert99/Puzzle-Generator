# Root Layout: Plain English Pseudocode

This document explains the core logic behind our `layout.tsx` file. In Next.js, the root layout is the foundational wrapper for your entire application. It breaks down the React and TypeScript syntax into plain English to help you understand *what* the code is doing and *why* it does it.

---

## 1. Setup & Imports

**Goal:** Pull in the necessary fonts, styles, and tools to build the shell of our website.
**Steps:**
1. Import the standard Next.js `Metadata` type to define our page's SEO settings.
2. Import two specific fonts from Google Fonts via Next.js's font optimization system: `Geist` (a sans-serif font) and `Geist_Mono` (a monospace font).
3. Import the global CSS stylesheet (`globals.css`) which contains our Tailwind configuration and custom styling rules.

---

## 2. Font Configuration

**Goal:** Configure our custom fonts so they can be injected into the website's CSS variables.
**Steps:**
1. Configure `Geist` (Sans): Assign it to the CSS variable `--font-geist-sans` and load the "latin" character subset.
2. Configure `Geist_Mono`: Assign it to the CSS variable `--font-geist-mono` and load the "latin" character subset.

---

## 3. Page Metadata (SEO)

**Goal:** Provide default information to web browsers and search engines.
**Steps:**
1. Export a `metadata` object containing the default title ("Create Next App") and description ("Generated by create next app") for the website. This information is injected into the `<head>` of the HTML document.

---

## 4. The Root Component

**Goal:** Create the main HTML wrapper that surrounds every single page in the application.
**Steps:**
1. Define the default exported function `RootLayout`. It takes a single prop called `children`, which represents whatever specific page the user is currently viewing.
2. Return the foundational HTML structure:
   - Create the `<html>` tag, set its language to English (`lang="en"`).
   - Inject the two font variables into the `className` of the `<html>` tag so they are available globally, and add Tailwind classes to make it fill the screen height (`h-full`) and smooth out the font rendering (`antialiased`).
   - Create the `<body>` tag. Add Tailwind classes to ensure it takes up at least the full height of the screen (`min-h-full`) and uses a flexible column layout (`flex flex-col`).
   - Finally, render the `children` (the actual page content) inside the `<body>`.
